### На чём сфокусировать UI / UX в MVP

*(приоритеты расставлены, чтобы «Pro‑лайт»‑аудитория получила ценность за первые 10 минут и вернулась снова)*

| Приоритет | Зона интерфейса                                            | Почему критично                                   | Конкретные задачи / Quick‑wins                                                                                                                                        |
| --------- | ---------------------------------------------------------- | ------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 🟥 1      | **“Первые 10 мин”**<br>Landing → Регистрация → Диагностика | Это весь activation‑фаннел; drop‑off здесь = \$ 0 | • Один CTA «Начать диагностику».<br>• Форма 1‑поля (е‑mail) + “Проверить позже”.<br>• Прогресс‑bar 0‑100 % + auto‑save.<br>• Skeleton‑лоадер ≤ 400 мс между экранами. |
| 🟥 2      | **Мостик (`/bridge`)**                                     | Даёт “aha‑moment”: радар + Next Steps             | • Радар + 3 квеста в первом view без скролла.<br>• Пустое состояние (no skills) → callout, ни курсов ни графиков.<br>• Анимация level‑up (0.3 c).                     |
| 🟥 3      | **Урок‑плеер (H‑E‑D‑T‑R)**                                 | Завершение 1‑го урока = retention                 | • 1 кнопка «Дальше», sticky‑footer.<br>• Try‑блок с AI‑помощником: input всегда в фокусе.<br>• «XP +25» toasts.<br>• `Esc` → выйти к Мостику.                         |
| 🟧 4      | **Навигация**                                              | Слишком много пунктов — когнитивный шум           | • Сайдбар: Мостик, Курсы, Лаб, blur‑demo для остального.<br>• Профиль в dropdown.<br>• Чёткие иконки (lucide), текст ≤ 10 симв.                                       |
| 🟧 5      | **Feedback loop**                                          | Пользователь должен видеть рост                   | • Инвалидировать `/skills/radar` после `POST /learning/progress`.<br>• Glow‑highlight сектора, XP‑ring sticky.                                                        |
| 🟨 6      | **Мобильная адаптация**                                    | 30‑40 % трафика с телефона                        | • Bridge — stack‑layout.<br>• Bottom‑tab вместо сайдбара.<br>• Урок‑плеер: swipe между этапами, кнопка «Голосовой ввод».                                              |
| 🟨 7      | **Пустые / Error‑states**                                  | Портят доверие                                    | • Callout вместо 404.<br>• «Сессия истекла» modal → логин.                                                                                                            |
| 🟩 8      | **Микро‑геймификация**                                     | Поддерживает внутренний драйв                     | • Конфетти при первом level‑up.<br>• Стрик‑огонёк на XP‑колесе.                                                                                                       |
| 🟩 9      | **Перфоманс**                                              | KPI Lighthouse ≥ 75 mobile                        | • Code‑split lessons (lazy).<br>• Изображения через `<Image priority />`.                                                                                             |

## Дорожная карта UI спринтов (4 недели)

| Спринт | Основной фокус                           | KPI приёмки                                 |
| ------ | ---------------------------------------- | ------------------------------------------- |
| 1      | Лёгкий Onboarding + Deep Diagnosis       | sign‑up → diag conversion ≥ 60 %            |
| 2      | Мостик с правильными пустыми состояниями | время до первого радара < 2 с               |
| 3      | Урок‑плеер + инвалидатор радара          | завершение урока ≥ 35 %                     |
| 4      | Mobile tweaks + blur‑demo меню           | TTI мобильный < 3 с; nav‑чистота (4 пункта) |

### Что **НЕ** делаем до запуска

* Голографический аватар‑тьютор (оставить text‑chat).
* Полную страницу Roadmap (placeholder достаточно).
* Сообщество / форум.
* NFT‑сертификаты.

Эти элементы не влияют на первые ключевые метрики (активация, урок‑completion) и могут дождаться post‑MVP.
